gh crfix

Hybrid GitHub PR review fixer — deterministic triage + AI gate + AI fix

bash script gh extension claude / codex parallel workers 
# install
gh extension install maszynka/gh-crfix

# run
gh crfix https://github.com/owner/repo/pull/123

▸ Deterministic triage

Every thread is classified before any LLM call. Only threads that cannot be handled deterministically reach the gate model.

⊘ skip

Thread needs no action and should not be resolved.

"LGTM" "Thanks!" "Looks good to me" question-only comments

⚙ auto (mechanical)

Simple deterministic fix — no reasoning required.

"nit: fix typo" "unused import on line 5" "eslint error here" "update changelog"

✓ already fixed

The code change requested is already present in the branch.

diff shows the fix was applied comment predates the latest push outdated thread on changed lines

🤖 needs LLM

Complex semantic review — requires model reasoning.

architectural feedback logic / correctness issues security concerns style beyond simple nits

▸ Gate score threshold

A numeric score determines whether the gate model runs at all. Each contributing factor adds its weight; the gate only fires when total ≥ 1.

Score contributors

Residual needs_llm threads +1.0
PR-level comment present +0.4
Deterministic validation failed +1.0
All weights tunable via flags or env vars.

Threshold

total ≥ 1

Below threshold → gate + fix models are skipped entirely. All deterministic responses are still posted.

Tune with --score-needs-llm --score-test-failure --score-pr-comment

Example: skip gate

needs_llm threads0
PR-level comment0
tests passed0
Total 0 → skip gate ✓

▸ Gate model → Fix model flow

needs_llm threads

residual after triage

→

Gate model

--gate-model sonnet

reads diff + threads + CI failures, decides if advanced model is worth running

→
needs_advanced_model: true
thread IDs selected
needs_advanced_model: false
post deterministic replies only
→

Fix model

--fix-model sonnet

edits files, commits, pushes — full access inside worktree

CI failure logs from failing GitHub Actions checks are passed to both the gate model and the fix model prompt.

PR #100
PR #101
PR #102
PR #103 waiting
PR #104 waiting
PR #105 waiting

Worker 1 — PR #100

✓ merge base ✓ triage (3 threads) ⟳ running gate model…

Worker 2 — PR #101

✓ merge base ✓ triage → 0 needs_llm → gate skipped (score 0)

Worker 3 — PR #102

⟳ merging base branch…
⚷ FIFO semaphore — max 3 concurrent workers; PR #103–105 block until a slot frees

▸ Isolated worktrees

What is a worktree?

Each PR gets its own git worktree — a separate directory on disk checked out to the PR branch. The AI model edits files there without touching your working directory.

  • Created at .gh-crfix/worktrees/pr-N/
  • Serialized creation (FIFO lock prevents races)
  • Base branch merged in before processing
  • Case-collision paths auto-normalized on macOS

What the fix model can do

Inside the worktree the model runs with full permissions:

  • Read and edit any file in the worktree
  • Run shell commands (tests, lint, build)
  • Create git commit with fixes
  • Push to the PR branch
  • Write thread-responses.json with planned replies

Use --dry-run to preview without any mutations.
